Site Audit App — Detailed Design
Mobile-first. Field conditions. Touch-forward. Every pixel earns its place.

  • 1
    Breadcrumb always visible
    Deep refrigeration hierarchy (Machine Room › Rack 2 › Compressor 1) shown in compact topbar — auditor always knows their position
  • 2
    One job per screen
    Each screen is one equipment item. Related fields visible together, but never two separate items on one screen
  • 3
    Photo auto-tagging
    Tap camera → capture → photo tagged to current equipment context automatically. Thumbnail shows tag confirmation immediately
  • 4
    Next button is the progress indicator
    When disabled, it shows "N required fields remaining" — the button itself communicates what's left without a separate error message
  • 5
    Connectivity: calm, not alarming
    Persistent banner replaces "Saved ✓" badge during reconnection. No modal, no data loss messaging. Data is always server-safe
  • 6
    Submit screen = certainty
    All 5 sections shown as green checkmarks with field counts. Auditor has unambiguous proof the audit is complete before submitting
Admin Console — Detailed Design
Command center for Star Energy engineers. Dense, efficient, state-aware. Every action is scoped to the current audit state.
Audit State Machine — Visual Pipeline
DRAFTAuditor working
IN_REVIEWClaude running calculations
FLAGGEDSLA paused · Admin review required
CALC_COMPLETEReady for admin approval
APPROVEDPublished to client portal
  • 1
    Dark sidebar = command center tone
    Navy `#111827` sidebar signals a professional, authoritative workspace. Raj is the authority here — not a form-filler
  • 2
    State-scoped actions only
    "Approve" only appears in CALC_COMPLETE rows. "Override" only in FLAGGED. No irrelevant buttons cluttering the interface
  • 3
    LLM flag = informative, not alarming
    Flag card shows Claude's reasoning and confidence score. Override requires typed justification — deliberate friction that protects audit quality
  • 4
    SLA clock visible and color-coded
    Green = time to spare. Amber = under 3 hours. Red = urgent. Paused SLA shown explicitly when audit is FLAGGED
  • 5
    Side panel for detail (not page navigation)
    Audit detail slides in from right without leaving the queue — Raj maintains queue context while drilling into a specific audit

  • 1
    Savings in human terms first
    $4,113/yr leads, not kWh. Sandra's CFO cares about dollars and CO₂ tonnes — not raw energy units. Energy reduction is secondary
  • 2
    Table of contents for navigation
    Reports can be long. TOC links let Sandra jump to Savings Projections without scrolling past equipment inventory she doesn't need right now
  • 3
    Methodology expandable, not foregrounded
    Transparent methodology builds trust, but Sandra's CFO doesn't want to read regression equations. Methodology section is available but collapsed by default
  • 4
    Approval attribution visible
    "Approved by Raj Patel · Star Energy" on every report — a credibility signal and a clear chain of accountability
  • 5
    PDF download is always present
    Sandra needs to share this with her CFO. PDF button is persistent in the report footer — one tap from anywhere in the report
  • 6
    Dashboard site cards = portfolio story
    Each card shows savings potential. Across 8 sites, the cumulative impact ($187K/yr) tells a portfolio story that justifies expanding the engagement
